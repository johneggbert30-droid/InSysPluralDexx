# ISPD7 System Hub

A customizable front-end hub for system profiles, headmates, chat, journaling, history, innerworld tracking, and account management.

## Front-end quick start

Open `index.html` in a browser, or use a local static server.

## Publish the front end with GitHub Pages

A GitHub Actions workflow (`.github/workflows/deploy.yml`) is included and will automatically deploy the `ISPD7/` folder to GitHub Pages on every push to `main`.

To enable it:

1. Push this repository to GitHub (or it will trigger on the next push to `main`).
2. In **Settings -> Pages**, choose:
   - **Source:** `GitHub Actions`
3. Save. The next push to `main` will publish the site and show the live URL in the Actions summary.

## Backend setup

A starter API is included in `backend/`.

```bash
cd backend
npm install
copy .env.example .env
npm start
```

Then update `config.js`:

```js
window.APP_CONFIG = {
  apiBaseUrl: 'http://localhost:3000',
  useBackendAuth: true
};
```

## Deploy backend

### Option 1 – Render (one-click)

A `render.yaml` is included at the repo root. Just:

1. Connect your GitHub repo to [Render](https://render.com).
2. Render will detect `render.yaml` and create the `ispd7-backend` service automatically.
3. Set the `FRONTEND_ORIGIN` environment variable to your GitHub Pages URL (e.g. `https://yourusername.github.io/your-repo-name`).
4. `JWT_SECRET` is auto-generated by Render.

### Option 2 – Other providers

You can also deploy the backend (`ISPD7/backend`) to Railway, Fly.io, or Azure App Service.
Copy `ISPD7/backend/.env.example` to `.env`, fill in the values, then follow the provider's Node.js deployment guide.

Once deployed, replace `apiBaseUrl` in `config.js` with the live backend URL and set `useBackendAuth: true`.
